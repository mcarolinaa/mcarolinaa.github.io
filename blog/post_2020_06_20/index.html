Important steps when you’re starting to put your hands into Data Science

I’m aware that once I get to actually work with ML and DS, things might change (and certainly sum up) from what I’ve written here. But it’s a start, and most important, it’s a start which I am truly feeling about how things should start to be done in a simple ML task.

I was inspired by this ‘division’ of ideas which I got from here (https://www.insidesherpa.com/virtual-internships/theme/m7W4GMqeT3bh9Nb2c/KPMG-Data-Analytics-Virtual-Internship)(Inside Sherpa - KPMG virtual internship). In this post I’ll cover part 1: Assessment of data quality.



1. Assessment of data quality

1.1 Importing data

First of all, you should start by acquiring your data.

I am assuming that your data is not extremely large, e.g., in a way that you wouldn’t be able to download in your machine.

Maybe the client sent you, maybe you’ve found it yourself. Anyway, let’s assume you’ll get the data and use it locally.

Data can be found in a variety of formats (.xls, .csv, .txt). Similarly, you can also choose to work with your data through several tools (R, python,..). Within each tool, there’s even more than one (sometimes much more than one) packages for you to choose and start working with your data.

Define a tool (language), check main packages to deal with the most common tasks of analyzing data, and try to keep some consistency - it may be easier for you to follow things this way, instead of jumping around packages that may have a different syntax.

Currently, I usually start using R, which I can do things more smoothly (less ‘googling’), but I’ve been trying to use python on a more frequent basis - both languages are great!

So, suppose I just started and I’m using R to import a dataset. I’ll use a dataset with information about penguins. More about this dataset can be found here: https://github.com/allisonhorst/palmerpenguins (also see references at the end of the post).

1.2 Check all datasets

It may be the case that you got a file with more than one dataset (i.e., more than one spreadsheet in a .xls or .csv). Open them all, take a look and confirm with your client (or the person you’re helping with the analysis) everything that you’re looking at.

Example: Suppose I just opened this .xls file, and I can see some spreadsheets, each one of them with a different name:

Probably each one has an important meaning that we should be aware of. It may even happen that one (or more) of them should’t be considered in the analysis. So, confirming all that you get and that you should use seems a good idea and may save you some time.

You don’t even have to use R or python to do this, a simple filter and other tools in the spreadsheets application will do.



1.3 Check all variables (structure, metadata, completeness, consistency, format)

Ok, so you confirmed with the client/friend and you have in your hands the data that was supposed to be with you to start the analysis.

Now you can start to actually ‘feel’ the data - and probably realize that you still don’t have everything you need to answer all questions!

Going back to the penguim dataset example. I did not get it from a file, it was directly downloaded into R, but let’s suppose I already check the data, and now I am going to put my hands on it.

# we'll check the simplified version of this dataset

library(palmerpenguins) # load dataset
library(tibble) # load package to deal with dataframes

glimpse(penguins) # take a look at the dataset: dimensions, head, types of variables

# Even if you have some categorical data, it is good to take a look at the statistical summary. We can deal with the categorical data later.
summary(penguins)

# How many NAs cells:
sum(is.na(penguins))

# Omit all NAs
penguins_clean <- na.omit(penguins)

Some relevant insights here: I have 4 numerical variables and 4 categoricals. No dates (that usually gives us some hard time to put into the right format).

By running the summary, I can see that I have few NAs in some columns. The cells with NAs represent a small portion (altough my dataset is small). If I omit all NAs, I’ll loose 10 rows. Let’s suppose this is fine for our purpose.

Other option to NAs would be to fill the variables with value like their average, for example. In this particular case, for the numerical cols, like body mass, with only 2 NAs, that could be fine.

But we also have more than 10 NAs on a categorical, sex, which I may want to treat as the target for future classification. This is one of the many situations in which you’ll see that it depends, each case should be analyzed separetely*.

Now it’s a good idea to take a closer look to data’s distributions. Let’s visualize some data.

1.4 Thinking step: what you understood about the dataset and how (and if) it can be used to answer questions (ex, the client’s questions)

After taking a closer look at the data, in terms of what you really have, what’s its structure, missing values and possible transformation, you can have a clearer idea of what can be done.

However, you need to combine these ‘thoughts’ with what the client expects/wants. I think this is a critical step!

Imagine that your client expects something unreal from the data, i.e., some prediction that would rely on data that they don’t have and it is unfeasible to get. Or, they do want something that will require you to research external data - that’s all important to be aware of!

To sum up, I think some important questions to bring to this conversation with yourself:

  • Can I provide insights on the desired questions with the data that I was given?

  • Will I have to research external data to give these insights? If so, is it possible to get these external data?

This is probably a phase where common-sense and awareness are very important. You have to be aligned with what you’re expected, and should be aware if that’s possible and be honest about what can and will be done.
